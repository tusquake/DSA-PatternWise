# Backend DSA Patterns

Common data structure and algorithm patterns used in backend development.

## Core Patterns

### 1. Hash Map / Hash Table
Fast lookups for caching, session management, and data indexing.

### 2. Queue
Task scheduling, message queues, job processing (RabbitMQ, Kafka).

### 3. Stack
Function call stack, undo operations, parsing expressions.

### 4. Linked List
LRU cache implementation, browser history.

### 5. Tree (Binary Search Tree)
Database indexing (B-trees, B+ trees), file systems.

### 6. Heap
Priority queues for job scheduling, finding top K elements.

### 7. Graph
Social networks, recommendation systems, routing algorithms.

### 8. Trie
Autocomplete, IP routing, prefix matching.

### 9. Sliding Window
Rate limiting, moving averages, time-based analytics.

### 10. Two Pointers
String manipulation, array processing, memory optimization.

### 11. BFS/DFS
Crawling systems, dependency resolution, graph traversal.

### 12. Binary Search
Searching sorted data, database queries, pagination.

### 13. Dynamic Programming
Resource allocation, optimization problems, caching strategies.

### 14. Sorting Algorithms
Data processing, ranking systems, log analysis.

### 15. Bloom Filter
Fast membership testing, caching, deduplication.

## Real-World Use Cases

- **Caching**: Hash Map, LRU Cache (Linked List + Hash Map)
- **Rate Limiting**: Sliding Window, Token Bucket
- **Load Balancing**: Consistent Hashing, Round Robin
- **Job Queues**: Priority Queue, FIFO Queue
- **Search**: Trie, Binary Search, Inverted Index
